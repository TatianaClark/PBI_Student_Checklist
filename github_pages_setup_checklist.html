🚀 GitHub Pages Setup Checklist

Follow these steps to publish your HTML checklist online for students and templates

0% Complete
📝 Tip: Use this checklist to guide you through creating shareable online checklists for your students and selling templates!

1Prerequisites

  • Have a GitHub account (sign up at github.com if needed)
  • Have Git installed on your computer
  • Have your HTML checklist file ready (power_bi_checklist.html)
  • Visual Studio or code editor installed

2Create GitHub Repository

  • Log in to your GitHub account
  • Click "+" icon → "New repository"
  • Name the repository (e.g., "student-checklists" or "checklist-templates")
  • Add a description of your checklists
  • Choose "Public" (so students/customers can access)
  • Click "Create repository"

3Push Files to GitHub

  • Open Visual Studio or Git Bash
  • Clone the repository: git clone [your-repo-url]
  • Copy your HTML checklist file(s) into the cloned folder
  • Create a README.md file to describe your checklists
  • Stage changes: git add .
  • Commit changes: git commit -m "Add checklist templates"
  • Push to GitHub: git push origin main

4Enable GitHub Pages

  • Go to your repository on GitHub
  • Click "Settings" tab
  • Scroll to "Pages" section in left menu
  • Select "main" branch as source
  • Click "Save"
  • Wait 1-2 minutes for deployment

5Access Your Published Checklists

  • Your site is live at: https://yourusername.github.io/repo-name/
  • Access specific checklist: https://yourusername.github.io/repo-name/filename.html
  • Test that all checklists load correctly
  • Verify checkboxes work and save properly

6Share with Students & Customers

  • Copy the public GitHub Pages URL
  • Share URL with students in course materials/emails
  • Create a landing page listing all available checklists
  • Set up a portfolio/sales page for template offerings
  • Add payment link (Gumroad, Stripe, etc.) for template sales

7Maintenance & Updates

  • Create new checklists as course progresses
  • Test updates locally before pushing
  • Push updates: git add . && git commit -m "message" && git push
  • Notify students of new checklist availability
  • Track student progress through course checklists
💡 Business Tip: You can use this same setup to create and sell specialized checklists on platforms like Gumroad, Etsy, or your own website. Each checklist can be a standalone product!
